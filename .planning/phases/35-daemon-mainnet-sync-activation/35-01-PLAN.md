# Plan 35-01: Sync Activation Config Contract

## Objective

Add a typed, disabled-by-default daemon sync activation contract that can be resolved from Open Bitcoin JSONC and daemon CLI arguments without accepting Open Bitcoin-only sync keys in `bitcoin.conf`.

## Requirements

- SYNCMAIN-01: `open-bitcoind` must expose explicit opt-in mainnet sync activation.
- SYNCMAIN-02: Config must keep public-network mainnet sync disabled by default.
- SYNCMAIN-04: Open Bitcoin-only settings must stay outside `bitcoin.conf`.
- RESUME-01: Durable runtime config must be restart-safe and explicit.
- VERMAIN-01: Activation must be testable without network access.
