<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| runtime adapters -> metrics contracts | Runtime telemetry will turn live node events into bounded metric samples. |
| runtime adapters -> log contracts | Runtime logging will turn local process events into structured files visible through status. |
| logs/status -> operator | Operators may inspect paths and recent health signals; secrets must not be exposed through contracts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-02-01 | D | `MetricRetentionPolicy` | mitigate | Encode exact bounded defaults of 30 seconds, 2880 samples, and 86400 seconds, and test those defaults. |
| T-13-02-02 | D | `LogRetentionPolicy` | mitigate | Encode daily rotation, 14 files, 14 days, and 268435456 bytes so rolling-file creation cannot be mistaken for retention. |
| T-13-02-03 | I | `logging.rs` | mitigate | Keep contracts limited to levels, paths, retention, and availability; do not model RPC passwords, cookie contents, or credential values as log/status fields. |
| T-13-02-04 | T | serialized metrics/logs | mitigate | Use serde rename rules and typed enums instead of free-form strings for metric kinds, metrics status, log levels, and log status. |
</threat_model>
