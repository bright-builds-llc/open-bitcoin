<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| live RPC/datadir/service collectors -> status snapshot | Later collectors will map local or live process state into this shared model. |
| status snapshot -> CLI/TUI/JSON renderers | Renderers and automation will trust stable field names and unavailable-state semantics. |
| build environment -> build provenance | Build metadata may be missing or supplied by release automation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-03-01 | I | `OpenBitcoinStatusSnapshot` | mitigate | Model credential sources only as paths/availability in later collectors; this plan's status model must not contain RPC password, cookie contents, or raw auth headers. |
| T-13-03-02 | T | `BuildProvenance` | mitigate | Require unavailable commit/build fields to serialize as `Unavailable { reason }`, preventing renderers from hiding missing provenance. |
| T-13-03-03 | R | stopped-node status | mitigate | Document and test that stopped nodes keep explicit unavailable reasons for live fields so support output can explain missing sync/peer/mempool/wallet data. |
| T-13-03-04 | S | daemon state | mitigate | Use typed `NodeRuntimeState` variants instead of free-form daemon-state strings. |
</threat_model>
