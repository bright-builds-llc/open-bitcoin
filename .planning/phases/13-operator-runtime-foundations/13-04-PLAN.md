## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| shell argv -> operator clap parser | Untrusted local command-line input enters the first-party operator command tree. |
| shell argv -> bitcoin-cli compatibility parser | Baseline-shaped RPC invocation tokens must remain interpreted by the existing parser. |
| operator docs/tests -> later command implementations | Later status/config/service/dashboard/onboard commands will implement this contract. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-04-01 | T | `route_cli_invocation` | mitigate | Route `open-bitcoin-cli` to `BitcoinCliCompat(Vec<OsString>)` without clap reinterpretation, and test preservation of `-named` tokens. |
| T-13-04-02 | I | clap help/output | mitigate | Do not model `rpcpassword`, cookie contents, or auth headers in the operator command contract; the compatibility parser retains existing auth handling. |
| T-13-04-03 | D | CLI parsing | mitigate | Keep `-stdin` and `-stdinrpcpass` in the compatibility parser path so normal no-stdin invocations cannot hang because of operator clap routing. |
| T-13-04-04 | S | binary route detection | mitigate | Add route tests for binary names ending in `open-bitcoin` and `open-bitcoin-cli` so the two command identities cannot silently collapse. |
</threat_model>
