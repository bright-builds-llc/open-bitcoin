## Threat Model

### Trust Boundaries
- **TB-02-03:** Untrusted serialized bytes -> first-party parsed structs.
- **TB-02-04:** First-party domain types -> serialized outputs later consumed by external peers or fixtures.

### Threat Register
- **Threat ID:** T-02-03
  - **Category:** Integrity
  - **Component:** CompactSize, witness flags, and transaction/block byte order
  - **Disposition:** mitigate
  - **Mitigation Plan:** Encode these rules explicitly in codec helpers and fail parsing on malformed or ambiguous byte streams.
- **Threat ID:** T-02-04
  - **Category:** Tampering
  - **Component:** network message headers and foundational payloads
  - **Disposition:** mitigate
  - **Mitigation Plan:** Keep message header parsing strict about field sizes and padding, and round-trip all supported payload structs through deterministic serializers.
</threat_model>
