# Phase 3.1: Legacy Signature Execution - Context

**Gathered:** 2026-04-11  
**Status:** Ready for planning and execution  
**Mode:** Yolo

<domain>
## Phase Boundary

Phase 3.1 finishes the legacy signature-validation slice that the Phase 3
foundation deferred. It owns legacy `CHECKSIG` and `CHECKMULTISIG` execution,
flag-gated signature or pubkey validation behavior, and `P2PKH` spending-path
verification through the canonical transaction-aware verifier. It does not own
P2SH, segwit-v0 witness programs, or taproot or tapscript execution.

</domain>

<decisions>
## Implementation Decisions

### Execution Shape
- **D-01:** Keep `verify_input_script` as the canonical transaction-aware
  spending-path verifier and extend it in place rather than introducing a
  second top-level execution API.
- **D-02:** Keep `eval_script` as the deterministic raw helper for non-signature
  unit tests, while adding an internal contextual evaluator for signature-aware
  legacy execution.

### Cryptography and Hashing
- **D-03:** Keep secp256k1 verification in the focused external crate already
  wired through Cargo and Bazel, but keep sighash, script, and pubkey-handling
  rules first-party in `open-bitcoin-consensus`.
- **D-04:** Add repo-owned `RIPEMD160` and `HASH160` support so `P2PKH`
  execution stays first-party and does not pull in Rust Bitcoin libraries.

### Legacy Rule Semantics
- **D-05:** Mirror the pinned Knots legacy behavior where signature or pubkey
  encoding failures become mandatory script errors only when the relevant flags
  are enabled; otherwise the signature check simply evaluates to false.
- **D-06:** Enforce `SIGPUSHONLY` before execution, keep the historical
  `CHECKMULTISIG` dummy-item bug, enforce `NULLDUMMY` and `NULLFAIL` in the
  same post-check cleanup paths as Knots, and leave witness-only flags deferred
  to later phases.

### Verification Strategy
- **D-07:** Port targeted upstream `script_tests.json` and `sighash.json`
  vectors that specifically exercise legacy `CHECKSIG`, `CHECKMULTISIG`,
  `P2PKH`, `DERSIG`, `LOW_S`, `STRICTENC`, `NULLDUMMY`, `NULLFAIL`, and
  sighash-mode behavior.
- **D-08:** Keep the repo-native `bash scripts/verify.sh` gate green after each
  slice instead of deferring coverage cleanup until the end of the phase.

</decisions>
