<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI/env/JSONC/bitcoin.conf -> config resolver | Untrusted operator input and local files become runtime config evidence. |
| Cookie files -> status/onboarding reports | Secret auth tokens must never cross into rendered output. |
| JSONC parser -> OpenBitcoinConfig | Invalid or unknown JSONC fields must fail deterministically. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-02-01 | Spoofing | source precedence | mitigate | Preserve exact source order in code and tests so lower-trust defaults cannot override explicit CLI/env choices. |
| T-17-02-02 | Tampering | JSONC and bitcoin.conf parsing | mitigate | Parse JSONC through `parse_open_bitcoin_jsonc_config`; continue to reject Open Bitcoin-only keys in `bitcoin.conf`. |
| T-17-02-03 | Information Disclosure | credential reporting | mitigate | Report only credential source and path availability; acceptance scans reject raw password, Authorization, Basic auth, and cookie contents. |
| T-17-02-04 | Denial of Service | malformed JSONC | mitigate | Return typed deterministic errors with the path/source instead of panicking or falling back silently. |
</threat_model>
