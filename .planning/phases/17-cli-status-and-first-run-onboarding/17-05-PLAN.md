<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Terminal/stdin -> onboarding shell | Interactive and non-interactive inputs can trigger file writes. |
| Runtime -> status RPC adapter | Live daemon evidence crosses from local HTTP RPC into shared status output. |
| Onboarding plan -> filesystem | Proposed JSONC contents cross into managed file writes. |
| Operator binary -> compatibility CLI | Wrong routing could break baseline-compatible `open-bitcoin-cli`. |
| Detection/status -> rendered output | Local paths and unavailable reasons become user-visible. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-05-01 | Tampering | onboarding JSONC writes | mitigate | Require `--approve-write` for new non-interactive writes and `--force-overwrite` for existing files; tests assert no overwrite without force. |
| T-17-05-02 | Tampering | bitcoin.conf compatibility | mitigate | Onboarding writes only `open-bitcoin.jsonc`; tests assert `bitcoin.conf` is not created or modified. |
| T-17-05-03 | Spoofing | binary/routing boundary | mitigate | Add `open-bitcoin` as a separate binary and keep `open-bitcoin-cli` tests green; service/dashboard return explicit phase-boundary messages. |
| T-17-05-04 | Information Disclosure | onboarding/status output | mitigate | Reuse status redaction rules and never print raw cookie contents, RPC passwords, Authorization headers, or wallet database contents. |
| T-17-05-05 | Denial of Service | non-interactive onboarding | mitigate | Missing required values fail deterministically with `missing required onboarding value` and do not prompt or write. |
| T-17-05-06 | Spoofing | live status adapter | mitigate | Running-node binary tests use deterministic fake RPC responses to prove adapter wiring, and RPC failures degrade to unavailable snapshot fields instead of fabricated live data. |
</threat_model>
