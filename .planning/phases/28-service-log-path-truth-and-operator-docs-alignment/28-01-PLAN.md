# Phase 28 Plan 01: Restore Service Log-Path Truth

## Objective

Repair the shared service runtime so launchd and systemd previews, apply paths,
and `service status` agree on one effective service log file path.

## Scope

- concrete service log-path derivation from operator config
- launchd and systemd generator updates
- installed-definition parsing for `service status`
- shared service-status rendering
- focused service-path tests

## Tasks

1. Add one shared helper that derives the concrete service log file path from
   the resolved operator log directory.
2. Update the service and dashboard runtime wiring to pass that concrete path
   through the shared execution path.
3. Extend launchd and systemd generators so both supported managers preserve the
   derived file path in generated artifacts.
4. Extend adapter status inspection and shared rendering so
   `open-bitcoin service status` always prints a `logs:` line with either the
   effective path or an explicit unavailable reason.
5. Prove the behavior with focused generator, parser, and service-command tests.
