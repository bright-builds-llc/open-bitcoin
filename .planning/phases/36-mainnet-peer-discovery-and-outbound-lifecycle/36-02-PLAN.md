# Plan 36-02: Bounded Outbound Peer Pool Runtime

## Objective

Refactor the durable sync runtime around a bounded outbound peer pool that tracks lifecycle state, retry/backoff, stall detection, and clean disconnect handling while reusing the existing pure-core peer/message engine.

## Requirements

- PEERMAIN-02: Maintain bounded outbound peer lifecycle state with timeout, retry backoff, stall detection, and clean disconnect handling.
- PEERMAIN-03: Prefer healthy peers and rotate unhealthy peers so one bad peer cannot block IBD indefinitely.
- RESUME-02: Invalid data, disconnects, and timeouts produce typed runtime failures and health signals.
- VERMAIN-02: Deterministic tests cover peer failures, resolver failures, and bounded-resource behavior.
