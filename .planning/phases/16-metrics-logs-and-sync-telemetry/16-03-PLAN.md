<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| remote peer/scripted transport -> sync runtime | Peer messages and connection failures influence sync summary, health signals, metrics, and logs. |
| sync runtime -> metrics storage | Runtime progress becomes durable metrics history. |
| sync runtime -> managed log directory | Runtime warnings/errors become structured JSONL records under a configured directory. |
| sync evidence -> later status/dashboard consumers | Phase 17 and Phase 19 will trust sync evidence without parsing raw network state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-03-01 | D | sync metrics persistence | mitigate | Use `append_metric_samples` with `MetricRetentionPolicy::default()` instead of overwrite-style snapshots, and test multi-run bounded history. |
| T-16-03-02 | T | peer-derived log records | mitigate | Encode through `StructuredLogRecord` and the managed JSONL writer; do not concatenate raw peer data into log files outside serde encoding. |
| T-16-03-03 | I | sync log messages | mitigate | Log concise source `sync` messages for stalls, retries, failures, and counters; do not include RPC credentials, cookie values, auth headers, or raw message payloads. |
| T-16-03-04 | R | retry/stall evidence | mitigate | Preserve actual retry attempt counts and emit `HealthSignal` values for stalled and failed peers so later status output can explain bottlenecks. |
| T-16-03-05 | D | default verification | mitigate | Keep telemetry tests hermetic with `ScriptedTransport`, temp stores, deterministic timestamps, and ignored live-network smoke coverage. |
</threat_model>
