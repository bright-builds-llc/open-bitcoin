<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| runtime collectors -> metrics core | Runtime values from sync, mempool, wallet, disk, RPC, and service collectors become numeric metric samples. |
| metrics core -> Fjall storage | Pure bounded history decisions cross into durable storage writes. |
| Fjall storage -> status consumers | Later CLI and dashboard consumers trust retained metrics history and availability metadata. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-01-01 | D | `append_and_prune_metric_samples` | mitigate | Enforce `MetricRetentionPolicy` max age and per-series sample caps before every durable save, with deterministic tests for both constraints. |
| T-16-01-02 | T | `MetricsStorageSnapshot` | mitigate | Persist only typed `MetricSample` values through the existing versioned snapshot codec and metrics namespace; do not write ad hoc keys or raw strings. |
| T-16-01-03 | I | metrics values | accept | Metrics are numeric runtime evidence, not secrets; unavailable collectors must be represented by `MetricsStatus` reason strings rather than fake samples or credential-bearing messages. |
| T-16-01-04 | R | status-facing metrics availability | mitigate | Add `load_metrics_status` tests proving missing metrics snapshots are reported as unavailable with a reason and with enabled-series metadata intact. |
</threat_model>
