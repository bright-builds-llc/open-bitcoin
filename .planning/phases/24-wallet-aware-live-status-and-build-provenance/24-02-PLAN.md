# Phase 24 Plan 02: Surface Real Build Provenance

## Objective

Populate the shared operator status model with real compile-time build metadata
when available, and make both status and dashboard render that provenance
explicitly.

## Scope

- Compile-time metadata emission for the CLI crate
- Shared snapshot assembly for build provenance
- Human status and dashboard rendering updates
- Focused build-provenance tests

## Tasks

1. Add a small Cargo build script that emits commit, build time, target, and
   profile metadata when they can be determined.
2. Replace hardcoded `BuildProvenance::unavailable()` calls in the status
   collector with a helper that consumes compile-time env vars and preserves
   unavailable reasons when absent.
3. Update the human status and dashboard build summaries to show the populated
   metadata fields instead of only version plus placeholder commit state.
4. Add focused tests for the build-provenance helper and for live status JSON
   output carrying populated build fields.
