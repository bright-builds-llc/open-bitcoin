# Phase 30 Plan 01: Repair Shared Live-RPC Bootstrap

## Objective

Restore the shared status/dashboard startup path so a missing implicit
`bitcoin.conf` no longer prevents truthful live RPC bootstrap for the
documented flag-only local operator workflow.

## Scope

- shared operator runtime startup derivation
- missing-implicit-config handling
- shared status/dashboard bootstrap preservation
- focused runtime and status regression tests

## Tasks

1. Extract or reshape a small helper in `packages/open-bitcoin-cli/src/operator/runtime.rs`
   so status/dashboard startup derivation comes from `OperatorConfigResolution`
   without hard-requiring the implicit `bitcoin.conf` path to exist.
2. Keep that helper precedence-aligned: always carry the selected datadir,
   include the selected network when present, and pass `-conf` only when the
   resolved `bitcoin.conf` path actually exists.
3. Continue using `resolve_startup_config()` as the final authority for live RPC
   endpoint/auth derivation, and keep `status_runtime_parts()` as the single
   shared bootstrap path for both `open-bitcoin status` and
   `open-bitcoin dashboard`.
4. Add focused regression tests that prove the missing-implicit-config case
   still yields a live bootstrap when datadir defaults are sufficient, while a
   genuine missing-credential or unreachable case still falls back cleanly.
