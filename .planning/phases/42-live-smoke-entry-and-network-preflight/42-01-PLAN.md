# Plan 42-01: Live Smoke Network Preflight And Typed No-Progress Evidence

## Objective

Extend the existing opt-in live-mainnet smoke runner so operators can pass
explicit manual peers, review DNS/manual-peer endpoint outcomes, preserve
cancellation as its own result, and get typed no-progress causes in the JSON and
Markdown reports without adding public-network access to default verification.

## Requirements

- PROOF-01: Operator can run an opt-in live-mainnet smoke command with explicit
  datadir, timeout, polling interval, and optional manual peers, and the command
  fails fast when local prerequisites are missing.
- PROOF-02: Operator can distinguish no-progress causes in the live smoke
  report, including DNS resolution failure, TCP connection failure, handshake
  failure, unsupported peer capability, validation failure, storage failure,
  timeout, and operator cancellation.
- PEER-01: Operator can preflight DNS seeds and manual peers and see which
  endpoints resolved, connected, handshook, failed, or were skipped before or
  during opt-in mainnet sync.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator CLI arguments -> generated config | Manual-peer inputs become daemon config used for public-network connection attempts. |
| Local network preflight -> report evidence | DNS/TCP diagnostics may influence operator troubleshooting and reviewer conclusions. |
| Runtime durable status -> typed diagnosis | Stored peer/status data is summarized into no-progress causes and next actions. |
| Cancellation signal -> partial report | Interrupt handling must stop the child process and still write accurate partial evidence. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-42-01-01 | Tampering | Generated JSONC config | mitigate | Generate the config from parsed manual-peer values only, record the generated path in JSON, and reject mixed `--config` plus `--manual-peer` input. |
| T-42-01-02 | Spoofing | Endpoint outcomes | mitigate | Mark each endpoint outcome with `stage` and `source` so preflight TCP checks are distinguishable from daemon runtime peer telemetry. |
| T-42-01-03 | Repudiation | No-progress diagnosis | mitigate | Persist `maybeNoProgressCause`, `nextAction`, endpoint outcomes, final status, and daemon tail in the report. |
| T-42-01-04 | Denial of Service | Network preflight | mitigate | Bound DNS/TCP preflight attempts and timeouts, and allow deterministic fixture injection for tests without public-network access. |
| T-42-01-05 | Information Disclosure | Reports | mitigate | Do not include RPC passwords beyond the existing local smoke command echo; manual peer data is operator-supplied public endpoint data. |
</threat_model>

## Success Criteria

- The smoke runner accepts explicit datadir, timeout, poll interval, output
  directory, disk floor, config path, and optional repeatable manual peers.
- Reports include machine-readable endpoint outcomes and Markdown tables for
  DNS/manual-peer preflight and durable runtime peer telemetry.
- No-progress reports carry a typed cause and concrete next action.
- Operator cancellation writes partial evidence with `status: cancelled`.
- Deterministic offline regression tests cover manual peers, endpoint outcomes,
  typed no-progress, and cancellation without public-network access.
