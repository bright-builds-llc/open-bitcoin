# Phase 11 Plan 01: Inventory And Classify Production Panic Sites

## Objective

Create a truthful inventory of first-party production panic-like sites and separate reachable crash risks from proven local invariants.

## Tasks

1. Scan first-party production Rust files for `unwrap`, `expect`, `panic!`, `unreachable!`, `todo!`, and `unimplemented!`.
2. Exclude vendored, generated, test-only, and inline `#[cfg(test)]` code from the classification.
3. Record classification categories in the guard allowlist: fixed-now, proven invariant, adapter/tooling acceptable, or false positive.

## Verification

- The scan is reproducible from a checked-in script.
- The classification can be reviewed without reading raw grep output.
