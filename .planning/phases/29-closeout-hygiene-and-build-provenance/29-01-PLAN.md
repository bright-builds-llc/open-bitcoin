# Phase 29 Plan 01: Restore Truthful Bazel Build Provenance

## Objective

Repair the Bazel build path so a Bazel-built `open-bitcoin status --format
json` surfaces truthful build provenance instead of the current `0.0.0` plus
all-`Unavailable` fallback.

## Scope

- Bazel workspace-status metadata
- shared CLI library target stamping
- compile-time version and provenance env wiring
- focused Bazel runtime provenance verification
- repo-native verification integration for the new check

## Tasks

1. Add a thin repo-owned Bazel workspace-status script that emits stable Git
   commit metadata.
2. Wire `.bazelrc` to that status script so Bazel stamping is available in local
   repo builds and verification.
3. Update `packages/open-bitcoin-cli/BUILD.bazel` so the shared CLI library
   target receives the workspace version plus truthful Bazel build metadata for
   commit, build time, target, and profile.
4. Add a focused Bun checker that runs the Bazel-built `open-bitcoin` binary,
   requests `status --format json`, and asserts the build-provenance fields are
   populated with truthful values.
5. Fold that focused check into `bash scripts/verify.sh` so the non-Cargo build
   path stays covered by the repo-native verification contract.

## Verification

- `bun run scripts/check-bazel-build-provenance.ts`
- `bash scripts/verify.sh`
