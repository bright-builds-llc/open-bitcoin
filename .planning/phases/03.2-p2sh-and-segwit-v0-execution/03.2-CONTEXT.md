# Phase 3.2: P2SH and Segwit-v0 Execution - Context

**Gathered:** 2026-04-12  
**Status:** Ready for planning and execution  
**Mode:** Yolo

<domain>
## Phase Boundary

Phase 3.2 extends the canonical transaction-aware verifier from the completed
legacy slice into `P2SH`, native segwit-v0, and nested segwit-v0 execution. It
owns redeem-script execution, witness-program handling, witness invariants, and
sigop accounting that depends on redeem or witness scripts. It does not own
taproot or tapscript behavior, and it does not broaden the pure-core consensus
surface beyond `P2SH`, `P2WPKH`, `P2WSH`, `P2SH-P2WPKH`, and `P2SH-P2WSH`.

</domain>

<decisions>
## Implementation Decisions

### Verifier Structure
- **D-01:** Keep `verify_input_script` as the single canonical spending-path
  verifier and extend it to follow the same control flow as Knots’
  `VerifyScript`: `scriptSig` then `scriptPubKey`, then bare witness programs,
  then `P2SH`, then nested witness, then final clean-stack or unexpected-witness
  checks.
- **D-02:** Keep `eval_script` as the raw deterministic helper and add only the
  minimal new opcode support required by the segwit-v0 surface, specifically
  control-flow support needed for `MINIMALIF` testing.

### Segwit-v0 Scope
- **D-03:** Support witness version 0 programs only in this phase. Unknown or
  future witness versions stay out of scope unless the existing Knots behavior
  is needed to preserve a soft-fork-compatible success or discouragement path.
- **D-04:** Implement both bare and nested segwit-v0 paths:
  `P2WPKH`, `P2WSH`, `P2SH-P2WPKH`, and `P2SH-P2WSH`.

### Rule Semantics
- **D-05:** Mirror the pinned Knots invariants where segwit-v0 spends require
  clean stack results, `P2SH` spends require push-only `scriptSig`, bare witness
  programs require empty `scriptSig`, nested witness spends require the
  `scriptSig` to be exactly a single push of the redeem script, and unexpected
  witness data is rejected when no witness program was actually consumed.
- **D-06:** Enforce `CLEANSTACK`, `MINIMALIF`, and `WITNESS_PUBKEYTYPE` in the
  segwit-v0 branches, while keeping taproot-only flags deferred to Phase 3.3.

### Sigop Accounting
- **D-07:** Keep the existing legacy sigop accounting for the context-free block
  slice, but add accurate `P2SH` and witness sigop counting to the contextual
  validation path so the split cost model matches Knots for the supported
  `P2SH` and segwit-v0 surface.
- **D-08:** Follow the Knots cost formula: legacy sigops scaled by witness
  factor, `P2SH` sigops scaled by witness factor, and witness-program sigops
  added directly.

### Verification Strategy
- **D-09:** Prefer focused first-party tests that mirror the upstream
  `VerifyScript`, `VerifyWitnessProgram`, and `CountWitnessSigOps` branches over
  broad imported harnesses in this phase; the larger parity corpus remains Phase
  3.4 work.

</decisions>
