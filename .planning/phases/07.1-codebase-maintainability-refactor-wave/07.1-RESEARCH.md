## Recommended Phase Order

1. Move inline tests from every oversized first-party file into sibling
   `foo/tests.rs` files, leaving production logic otherwise intact.
2. Split `packages/open-bitcoin-wallet/src/wallet.rs` into `wallet.rs` plus
   `wallet/build.rs`, `wallet/scan.rs`, and `wallet/sign.rs` so the public API
   remains readable and the implementation seams match existing responsibilities.
3. Split `packages/open-bitcoin-consensus/src/script.rs` into `script.rs` plus
   child modules for opcodes, stack or numeric helpers, instruction decoding,
   legacy execution, witness or taproot handling, and sigop counting.

## Minimum Phase Truths

1. Oversized first-party Rust files no longer mix large inline unit suites into
   production source where the tests can live in sibling modules instead.
2. `wallet.rs` retains the same public surface while its scan, build, and sign
   responsibilities move into coherent child files.
3. `script.rs` retains the same public verifier and sigop surface while its
   implementation details move into child files and the consensus corpus still
