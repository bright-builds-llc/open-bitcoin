<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| RPC/local filesystem -> status snapshot | Live or stopped-node evidence becomes support output. |
| Detection results -> status snapshot | Core/Knots candidate paths and uncertainty become support evidence inside shared health signals. |
| Status snapshot -> human/JSON renderers | Potentially sensitive paths and auth metadata become operator-visible. |
| Log/metrics stores -> CLI | Local evidence can be missing, corrupt, or stale. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-04-01 | Information Disclosure | status JSON/human output | mitigate | Redaction tests reject raw passwords, Authorization/Basic headers, and cookie contents; report only credential source/path. |
| T-17-04-02 | Denial of Service | RPC/status collector | mitigate | RPC failures produce unavailable fields with reasons instead of failing the whole status command. |
| T-17-04-03 | Tampering | metrics/log local evidence | mitigate | Use Phase 16 loaders; do not parse unmanaged raw files outside configured managed paths. |
| T-17-04-04 | Spoofing | build/config/source reporting | mitigate | Include source paths, build provenance, and unavailable reasons so support output does not imply evidence was collected when it was not. |
| T-17-04-05 | Spoofing | Core/Knots detection evidence | mitigate | Map `DetectedInstallation` confidence and uncertainty text into health signals; tests assert uncertain candidates are not rendered as definitive migration decisions. |
</threat_model>
