# Phase 31 Plan 01: Scope Service Review To The Selected Source

## Objective

Keep migration service cutover review truthful by associating service-definition
evidence to the selected source installation before the planner renders the
service action group or source summary.

## Scope

- Migration planner service-association logic only
- Focused migration unit and operator-binary regressions

## Tasks

1. Add a small migration-scoped helper that reads launchd or systemd
   service-definition arguments and extracts source ownership evidence from
   datadir or config-path flags.
2. Use that helper so the selected-source summary, service action group, and
   service-surface deviation relevance only treat matched service definitions as
   source-specific.
3. Preserve an explicit manual-review fallback when service evidence exists but
   cannot be confidently tied to the selected install.
4. Add focused tests for both matched-source inclusion and ambiguous-service
   fallback, and tighten the custom-source binary regression so it no longer
   accepts unrelated service review paths.

- `cargo test --manifest-path packages/Cargo.toml -p open-bitcoin-cli --test operator_binary -- --nocapture open_bitcoin_migrate_plan_selects_explicit_custom_source_outside_default_roots`
