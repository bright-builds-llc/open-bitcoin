# Phase 24 Plan 04: Normalize Build-Time Provenance Format

## Objective

Close the Phase 24 UAT gap where Cargo status JSON reports
`build.build_time.value` as an ISO-8601 UTC timestamp but Bazel status JSON
reports epoch seconds. Standardize on ISO-8601 UTC for all operator-facing build
provenance.

## Scope

- Cargo build-time formatting
- Bazel workspace-status metadata
- Bazel provenance regression check
- UAT gap closure bookkeeping
