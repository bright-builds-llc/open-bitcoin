# Phase 34 Plan 01: Introduce The Scan-Level Detection Ownership Model

## Objective

Repair the shared read-only detection model so scan-wide service definitions are
returned as scan-level evidence instead of being copied into every
`DetectedInstallation`.

## Scope

- Shared detection types and detector tests
- Runtime threading and status fallback adoption
- Nearby fixture updates required by the tightened installation model

## Tasks

1. Introduce a dedicated scan-level detection type in `operator/detect.rs` that
   carries detected installations and scan-wide service candidates separately.
2. Remove `service_candidates` from `DetectedInstallation` and tighten detector
   construction so installation-local evidence remains local.
3. Update runtime and status collection to pass the new scan-level evidence
   through the shared command path without regressing fallback service truth.
4. Update detector, status, dashboard, onboarding, and wallet fixtures so they
   stop constructing installation-local service candidates that no longer exist.
