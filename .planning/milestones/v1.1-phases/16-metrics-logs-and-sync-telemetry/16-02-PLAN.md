| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-02-01 | T | JSONL log records | mitigate | Encode records with `serde_json::to_vec` instead of string concatenation; read records through typed `StructuredLogRecord` deserialization. |
| T-16-02-02 | D | log retention | mitigate | Enforce `LogRetentionPolicy` `max_files`, `max_age_days`, and `max_total_bytes` with pure planner tests and adapter tests. |
| T-16-02-03 | T | log pruning | mitigate | Delete only files matching `open-bitcoin-runtime-<unix_day>.jsonl` discovered under the configured log directory; unmanaged names are never selected by the planner. |
| T-16-02-04 | I | log messages and health signals | mitigate | Keep records source-scoped and concise; explicitly forbid serialization of RPC cookies, passwords, tokens, auth headers, and raw request payloads. |
| T-16-02-05 | R | missing log paths | mitigate | `load_log_status` returns explicit unavailable path status with a reason, preserving retention metadata for stopped-node inspection. |
