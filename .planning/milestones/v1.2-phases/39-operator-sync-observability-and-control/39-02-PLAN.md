## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator CLI -> local daemon RPC | User-controlled CLI process sends sync-control requests over HTTP to the daemon. |
| daemon RPC -> sync worker control channel | Authenticated RPC dispatch crosses into daemon-owned mutable sync control. |
| daemon sync worker -> Fjall store | The worker persists pause/resume and sync status to the selected datadir. |
| fake/local RPC test -> CLI renderer | Test server responses are untrusted JSON and must be parsed through typed response handling. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-39-02-01 | Spoofing | `packages/open-bitcoin-cli/src/operator/runtime/support.rs` live RPC client | mitigate | Use the existing RPC auth config and `Authorization` header helpers. Treat HTTP 401/403 or JSON-RPC auth errors as terminal failures; never fall back to direct-store writes after auth failure. |
| T-39-02-02 | Tampering | `openbitcoinsyncpause` / `openbitcoinsyncresume` RPC handlers | mitigate | Keep handlers node-scoped, authenticated by the existing daemon HTTP layer, and route mutations through the daemon sync control port so only the daemon-owned runtime writes `RuntimeMetadata.sync_control`. |
| T-39-02-03 | Repudiation | operator pause/resume control | accept | Phase 39 already records durable sync state and structured sync logs; this gap closure does not add a new audit log requirement. Tests must still prove pause/resume state is persisted and observable. |
| T-39-02-04 | Information Disclosure | CLI/RPC error rendering | mitigate | Do not print `rpcpassword`, cookie contents, or Authorization headers in live sync-control errors. Reuse existing status/wallet RPC client redaction patterns. |
| T-39-02-05 | Denial of Service | daemon sync worker control channel | mitigate | Service sync-control commands between bounded sync rounds and keep CLI RPC timeouts finite so a stuck daemon returns an actionable error instead of hanging indefinitely. |
| T-39-02-06 | Elevation of Privilege | offline fallback path | mitigate | Permit direct-store fallback only when RPC is unreachable or not configured. A reachable daemon error must not downgrade to local store mutation, preventing auth bypass and the original lock failure. |
